# BEI-Analysis
# README

## Project title
How Robust Are Bridge-Based Conclusions? Partition Dependence of Bridge Identification in an Internalizing Symptom Network

## Overview
This repository contains the analysis code used to reproduce the main results, tables, figures, and supplementary analyses reported in the manuscript.

The project evaluates whether bridge-based conclusions derived from Bridge Expected Influence (BEI) are robust to alternative community partitions in an internalizing symptom network based on 35 SCL-90-R items.

The code reproduces:
- network estimation
- community detection and partition comparison
- bridge expected influence analyses
- node-knockout simulations
- internal proxy performance analyses
- temporal stability analyses
- supplementary sensitivity analyses

## Repository contents
- `01_data_prep.R`  
  Data cleaning, item selection, and quality-control filtering.

- `02_network_estimation.R`  
  Estimates the primary EBICglasso network and computes network descriptives.

- `03_partition_comparison.R`  
  Runs Walktrap and Louvain community detection and computes ARI-based partition comparisons.

- `04_bridge_analysis.R`  
  Computes Bridge Expected Influence under empirical and a priori partitions.

- `05_knockout_analysis.R`  
  Performs node-removal simulations and bridge-versus-control comparisons.

- `06_internal_proxy.R`  
  Computes the Sentinel short-form analysis and benchmark comparisons.

- `07_temporal_stability.R`  
  Re-estimates networks by period and computes temporal concordance metrics.

- `supplementary_analyses.R`  
  Generates supplementary tables and robustness checks.

- `figures/`  
  Output figures generated by the scripts.

- `tables/`  
  Output tables generated by the scripts.

## Data availability
The raw data used in this project were collected as part of routine institutional mental health screening and cannot be shared publicly because of privacy and institutional governance restrictions.

To support reproducibility, this repository includes:
- analysis code
- variable definitions
- derived summary outputs
- table and figure generation code

If a de-identified or synthetic demonstration dataset is provided, place it in the `data/` folder.
